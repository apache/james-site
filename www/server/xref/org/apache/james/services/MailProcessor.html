22  /***
23   * NOTE: this is an experimental interface. It will be probably dropped later.
24   * 
25   * @since 2.3.0a3
26   * 
27   * The <code>service</code> perform all needed work
28   * on the Mail object. Whatever remains at the end of the service is considered
29   * to need futher processing and will go to the next Mailet if there is one
30   * configured or will go to the error processor if not.
31   * Setting a Mail state (setState(String)) to Mail.GHOST or cleaning its recipient
32   * list has the same meaning that s no more processing is needed.
33   */
34  public interface MailProcessor {
35      /***
36       * Called by the mailet container to allow the mailet to process to
37       * a message.
38       * <p>
39       * This method is only called after the mailet's init() method has completed
40       * successfully.
41       * <p>
42       * Mailets typically run inside multithreaded mailet containers that can handle
43       * multiple requests concurrently. Developers must be aware to synchronize access
44       * to any shared resources such as files, network connections, as well as the
45       * mailet's class and instance variables. More information on multithreaded
46       * programming in Java is available in <a href="http://java.sun.com/Series/Tutorial/java/threads/multithreaded.html">the
47       * Java tutorial on multi-threaded programming</a>.
48       *
49       * @param mail - the Mail object that contains the message and routing information
50       * @throws javax.mail.MessagingException - if a message or address parsing exception occurs or
51       *      an exception that interferes with the mailet's normal operation
52       */
53      void service(Mail mail) throws javax.mail.MessagingException;
54  }
